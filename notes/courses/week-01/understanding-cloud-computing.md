# Course: Understanding Cloud Computing

# 1. Summary & Goals

**Course page:** [Understanding Cloud Computing](https://www.datacamp.com/courses/understanding-cloud-computing)

Understanding Cloud Computing is a beginner-level, non-coding course that establishes a provider-neutral foundation for reasoning about cloud infrastructure. It explains how cloud hosting differs from on-premises hosting, why organizations adopt cloud services, how responsibility is divided across service models, how deployment choices affect control and scalability, and how major providers differ.

## Course Promise

Build a provider-neutral mental model of cloud computing so that hosting models, service models, deployment models, governance constraints, and provider choices can be distinguished and evaluated coherently.

## Learning Goals

By the end of the course, the learner should be able to:

- Define cloud computing and distinguish cloud-based hosting from on-premises infrastructure.
- Explain why scalability, availability, latency, collaboration, disaster recovery, and usage-based costs influence cloud adoption.
- Distinguish Infrastructure as a Service, Platform as a Service, and Software as a Service by provider and consumer responsibility.
- Distinguish public, private, and hybrid cloud deployments by control, scalability, security, and cost.
- Recognize how security, regulation, personal data, and data location constrain cloud decisions.
- Identify the organizational roles involved in planning, operating, and securing cloud infrastructure.
- Compare the broad positioning and services of AWS, Microsoft Azure, Google Cloud, and other providers.
- Evaluate cloud-provider and service choices using organizational requirements rather than provider popularity alone.

## Chapter 3: Cloud Providers and Case Studies

_Applies the earlier cloud models to the provider landscape, examining the positioning, services, customers, and practical use cases of AWS, Microsoft Azure, Google Cloud, and other providers._

- Cloud providers expose broadly similar capabilities—compute, storage, databases, analytics, streaming, and machine learning—under different product names.  
- Provider selection should begin with organizational requirements, existing systems, migration effort, staffing, total cost, security, regulation, and operational risk rather than market share alone.  
- AWS, Microsoft Azure, and Google Cloud were examined through representative services and customer cases.  
- Managed services can remove specific workflow bottlenecks, but choosing a major provider does not automatically reduce cost or improve reliability.  
- Cloud adoption may replace an environment, extend an existing environment, or support one targeted capability such as disaster recovery.  
- Scalability and availability depend on the deployed architecture and configuration; moving to the cloud is not itself a guarantee.
